# 身分組：比奇堡後端組

## 概要

| 項目 | 內容 |
|------|------|
| 身分組名稱 | 比奇堡後端組 |
| 成員 | 章魚哥（PM）、派大星（後端工程師）、泡芙老師（Code Reviewer） |
| 觸發條件 | 人類在蟹堡王/會議室 mention 章魚哥交辦後端需求 |
| 主要窗口 | 章魚哥 |
| 最終回報對象 | 第一個發言的人類（即觸發需求的人） |

## 流程總覽

```
人類發話（需求）
    ↓
章魚哥（接收、釐清、產規格）
    ↓ 指派
派大星（開發、開 PR）
    ↓ mention 泡芙
泡芙老師（review）
    ↓ 通過後 mention 章魚哥
章魚哥（回報發話人：任務完成 + PR 連結）
    ↓
人類接手（確認、測試、合併）
```

## 角色職責

### 章魚哥（主要窗口 / PM）

- 接收人類需求，釐清規格
- **記住「發話人」**：thread 中第一個提出需求的人類，即為本次任務的回報對象
- 規格完成後直接指派派大星，不需要額外找珈瑄確認
- 收到泡芙老師的 review 通過通知後，回報發話人

### 派大星（後端工程師）

- 收到章魚哥指派後開發
- 完成後開 PR，mention 泡芙老師 review
- review 不通過 → 修正後再次 mention 泡芙
- review 通過後 → 泡芙會通知章魚哥，派大星任務結束

### 泡芙老師（Code Reviewer）

- 收到派大星的 review 請求後審閱
- 不通過 → mention 派大星修正
- 通過 → mention 章魚哥（而非直接通知人類審核者）

|----------|-----------|-------------|

| 規格不清需追問 | 發話人 | 依發話人 UID |

### 派大星

|----------|-----------|-------------|

### 泡芙老師

|------|-----------|-------------|
| 不通過 | 派大星（修正） | `<@1496023645083009024>` |
| 通過 | 章魚哥（回報流程） | `<@1503698574477627482>` |
| 第 3 輪仍有爭議 | 章魚哥（決定後續） | `<@1503698574477627482>` |

## 「發話人」判斷規則

1. 在 thread 中，第一個 mention 章魚哥並提出需求的**人類**（非 bot）
2. 章魚哥在 `_status.md` 中記錄該任務的發話人 UID
3. 整個任務生命週期中，回報對象不變
4. 如果發話人不明確 → 章魚哥主動詢問「這個任務完成後要回報給誰？」

## 與原有流程的差異

| 項目 | 原流程 | 後端組流程 |
|------|--------|-----------|
| 規格確認 | 章魚哥 → 珈瑄確認 → 指派 | 章魚哥直接指派（不需珈瑄確認） |
| PR review 通過後 | 泡芙 → 詠仁/珈瑄 | 泡芙 → 章魚哥 → 發話人 |
| 最終回報對象 | 固定（珈瑄/詠仁/潔庭） | 動態（第一個發話的人類） |
| 卡住時找誰 | 派大星 → 珈瑄 | 派大星 → 章魚哥 |

## 適用場景

- 人類（任何有權限的人）在蟹堡王或會議室交辦後端開發任務
- 章魚哥判斷為後端需求且指派派大星時，自動進入此流程
- 不適用於：hotfix（仍走原流程）、前端任務（海綿寶寶走另一個身分組）
