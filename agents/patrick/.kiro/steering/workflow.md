# 工作流程

## 專案管理

- 工作目錄：`/home/agent/projects`
- 維護 `_projects.md`：記錄所有接手過的專案（路徑、repos、關鍵字）
- 每個 repo 內維護 `_status.md`：當前任務、最近完成（最多 5 筆）、待確認項目
- 超過 5 筆的完成紀錄移到同目錄的 `_archive.md`，平時不讀

## 新 Session 啟動

1. 讀 `_projects.md` → 從訊息比對關鍵字確定專案
2. 進入該專案目錄，讀對應 repo 的 `_status.md`
3. **如果「待交棒」有未完成項目 → 立即執行 mention，不做其他事**
4. 如有 specs/ 目錄，`git pull` 取最新規格
5. 比對不到專案 → 問對方是哪個專案
6. 全新專案 → clone repo，在 `_projects.md` 新增紀錄

## 任務完成後

1. 判斷是否需要交棒（參考交棒規則）
2. 如需交棒 → 在 `_status.md` 的「待交棒」區塊寫入：要 mention 誰、原因
3. 執行 mention
4. mention 成功後 → 從「待交棒」移除
5. 更新「最近完成」
6. 「最近完成」超過 5 筆 → 最舊的移到 `_archive.md`

## 何時讀 `_archive.md`

- 有人問過去的任務細節
- 需要參考過去的實作方式
- 被明確要求回顧歷史

## Redmine 任務處理

收到 Redmine issue 時：
1. 讀 issue → 確認理解 → 在 Discord 簡述處理計畫
2. 等確認後開始 → 更新 issue 狀態為開發中（status_id:2, 10%）
3. 執行任務 → 完成後在 issue 留言回報
4. 狀態改為 review（status_id:9, 90%）

## 泡芙老師通過後 — 到電視台發 PR 請求

泡芙老師通知你 review 通過後，用 Discord API 主動推訊息到 📺 鯡魚電視台，並開出 thread：

```bash
# Step 1: 發訊息到電視台頻道
MSG_RESPONSE=$(curl -s -X POST "https://discord.com/api/v10/channels/${CHANNEL_TV}/messages" \
  -H "Authorization: Bot ${DISCORD_BOT_TOKEN}" \
  -H "Content-Type: application/json" \
  -d "{\"content\": \"<@1505819268384817313> PR 請求審核，泡芙老師已通過 ✅\n\nPR 連結：${PR_URL}\n簡述：${PR_SUMMARY}\"}")

# Step 2: 取得訊息 ID，開出 thread
MSG_ID=$(echo "$MSG_RESPONSE" | jq -r '.id')
curl -s -X POST "https://discord.com/api/v10/channels/${CHANNEL_TV}/messages/${MSG_ID}/threads" \
  -H "Authorization: Bot ${DISCORD_BOT_TOKEN}" \
  -H "Content-Type: application/json" \
  -d "{\"name\": \"PR Review: ${PR_TITLE}\"}"
```

**注意**：`DISCORD_BOT_TOKEN` 和 `CHANNEL_TV` 已在環境變數中，直接使用即可。
