# 工作流程

## 專案管理

- 工作目錄：`/home/agent/projects`
- 維護 `_projects.md`：記錄所有接手過的專案（路徑、repos、關鍵字）
- 每個 repo 內維護 `_status.md`：當前任務、最近完成（最多 5 筆）、待確認項目
- 超過 5 筆的完成紀錄移到同目錄的 `_archive.md`，平時不讀

## 新 Session 啟動

1. 讀 `_projects.md` → 從訊息比對關鍵字確定專案
2. 進入該專案目錄，讀對應 repo 的 `_status.md`
3. 如有 specs/ 目錄，`git pull` 取最新規格
4. 比對不到專案 → 問對方是哪個專案
5. 全新專案 → clone repo，在 `_projects.md` 新增紀錄

## 任務完成後

1. 更新對應 repo 的 `_status.md`
2. 「最近完成」超過 5 筆 → 最舊的移到 `_archive.md`

## 何時讀 `_archive.md`

- 有人問過去的任務細節
- 需要參考過去的實作方式
- 被明確要求回顧歷史

## Redmine 任務處理

收到 Redmine issue 時：
1. 讀 issue → 確認理解 → 在 Discord 簡述處理計畫
2. 等確認後開始 → 更新 issue 狀態為開發中（status_id:2, 10%）
3. 執行任務 → 完成後在 issue 留言回報
4. 狀態改為 review（status_id:9, 90%）

## Discord 回覆原則

- 只講重點：結果、需確認事項、遇到的問題
- 背景作業（讀檔案、更新狀態）不需要說明
